Week 13: JS DOM Events & Forms – Part 1

This week focuses on events, form inputs, validations, and interactive pages. Each day includes examples, exercises, and checks to ensure understanding.

Day 1: Click Events

Learn: Click events let elements respond to user clicks. You can use onclick or addEventListener.

<button id="btn">Click Me</button>
<script>
document.getElementById("btn").addEventListener("click", function() {
  alert("Button clicked!");
});
</script>

Do: Create a button that shows your name in an alert when clicked.

Check: Click the button in your browser; the alert should show your name.

Tip: addEventListener is preferred over onclick for cleaner, maintainable code.
Day 2: Input Fields & Getting Values

Learn: Inputs allow users to enter data, and JS can access it using .value.

<input id="nameInput" placeholder="Enter name">
<button id="showBtn">Show Name</button>
<script>
document.getElementById("showBtn").addEventListener("click", function() {
  let name = document.getElementById("nameInput").value;
  alert("Hello " + name);
});
</script>

Do: Add an input for your favorite hobby and display it in an alert.

Check: Enter text and click the button to verify the alert shows the correct value.

Tip: Always read input values using .value and test empty inputs.
Day 3: Input Events – Live Updates

Learn: input and change events let you respond as users type or finish input.

<input id="liveInput" placeholder="Type something">
<p id="display"></p>
<script>
document.getElementById("liveInput").addEventListener("input", function() {
  document.getElementById("display").innerText = "You typed: " + this.value;
});
</script>

Do: Display live feedback as the user types in an input box.

Check: Open in browser and type; the paragraph should update instantly.

Tip: Use input for instant feedback, change triggers only when leaving the field.
Day 4: Form Submission & Validation

Learn: Validate input before submission to prevent errors.

<form id="signupForm">
  <input id="username" placeholder="Username">
  <input id="age" type="number" placeholder="Age">
  <button>Submit</button>
</form>

<script>
document.getElementById("signupForm").addEventListener("submit", function(e) {
  let username = document.getElementById("username").value;
  let age = document.getElementById("age").value;
  if(username === "" || age === "" || age < 0) {
    alert("Enter a valid username and age");
    e.preventDefault();
  } else {
    alert("Welcome " + username + "!");
  }
});
</script>

Do: Create a form asking for name and age; prevent submission if empty or invalid.

Check: Try submitting empty or negative age; the alert should block submission.

Tip: Use event.preventDefault() to stop form submission for invalid data.
Day 5: Mouse Events

Learn: Mouse events like mouseover, mouseout, mousedown, mouseup let you respond to user interaction on elements.

<div id="box" style="width:100px;height:100px;border:1px solid #000"></div>
<script>
let box = document.getElementById("box");
box.addEventListener("mouseover", () => box.style.background = "lightblue");
box.addEventListener("mouseout", () => box.style.background = "white");
box.addEventListener("mousedown", () => box.style.border = "2px solid red");
box.addEventListener("mouseup", () => box.style.border = "1px solid black");
</script>

Do: Create a div that changes color on hover and border when clicked.

Check: Mouse over and click the box to verify changes.

Tip: Mouse events are great for interactive effects like hover highlights, buttons, and visual feedback.
Day 6: Keyboard Events

Learn: Detect key presses using keydown, keyup, or keypress.

document.addEventListener("keydown", function(e) {
  console.log("Key pressed: " + e.key);
});
<!-- Optional: highlight an input when a specific key is pressed -->

Do: Log all letters pressed in the console; optionally show a message when Enter is pressed.

Check: Press keys in the browser; console logs each key.

Tip: Use e.key for the character and e.code for the physical key location.
Day 7: Review & Interactive Mini Page

Learn: Combine inputs, events, mouse/keyboard interactions, and validation for a fully interactive page.

Do: Build a mini sign-up form: ask for name and age, update messages live as user types, validate on submit, and add a colored div that reacts to mouse hover.

Check: Open in browser, test typing, submit valid/invalid input, and hover/click interactions. Ensure all feedback and validations work correctly. This completes Week 13.

Tip: Combining event listeners, DOM manipulation, and validation creates engaging and responsive pages.
