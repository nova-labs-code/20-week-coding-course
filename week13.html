Week 13: JS DOM Events & Forms – Part 1

This week teaches events, form inputs, and basic interaction to make dynamic pages that respond to user actions.

Day 1: Click Events Review

Learn: Buttons and elements can react to clicks using addEventListener.

<button id="btn">Click Me</button>
<script>
document.getElementById("btn").addEventListener("click", function() {
  alert("Button clicked!");
});
</script>

Do: Create a button that shows your name in an alert on click.

Tip: Use addEventListener instead of onclick for cleaner code.
Day 2: Input Fields

Learn: HTML inputs allow users to type data that JS can access.

<input id="nameInput" placeholder="Enter name">
<button id="submitBtn">Submit</button>
<script>
document.getElementById("submitBtn").addEventListener("click", function() {
  let name = document.getElementById("nameInput").value;
  alert("Hello " + name);
});
</script>

Do: Add an input for your favorite hobby and display it in an alert.

Tip: Always read values with .value for text inputs.
Day 3: Change & Input Events

Learn: Input fields can react live as users type using oninput or addEventListener("input").

<input id="textInput">
<p id="display"></p>
<script>
document.getElementById("textInput").addEventListener("input", function() {
  document.getElementById("display").innerText = this.value;
});
</script>

Do: Display typed text live in a paragraph.

Tip: input updates instantly; change fires only after leaving the field.
Day 4: Form Validation

Learn: JS can validate input before form submission.

<form id="form">
  <input id="age" type="number">
  <button>Submit</button>
</form>

<script>
document.getElementById("form").addEventListener("submit", function(e) {
  e.preventDefault();
  let age = document.getElementById("age").value;
  if(age === "" || age < 0) alert("Enter a valid age");
  else alert("Age entered: " + age);
});
</script>

Do: Create a form asking for username and prevent submission if empty.

Tip: Return false in the validation function to block submission.
Day 5: Mouse Events

Learn: Elements can respond to mouseover, mouseout, mousedown, mouseup.

<div id="box" style="width:100px;height:100px;border:1px solid #000"></div>
<script>
let box = document.getElementById("box");
box.addEventListener("mouseover", () => box.style.background = "lightblue");
box.addEventListener("mouseout", () => box.style.background = "white");
</script>

Do: Create a div that changes color on hover.

Tip: Mouse events are useful for interactive effects like hover highlights or tooltips.
Day 6: Keyboard Events

Learn: Detect key presses with keydown or keyup.

document.addEventListener("keydown", function(e) {
  console.log("Key pressed: " + e.key);
});

Do: Log letters pressed in console.

Tip: Use e.key to get the actual character, e.code for the physical key.
Day 7: Review & Interactive Form

Learn: Combine inputs, events, and validation for interactive pages.

Do: Build a mini sign-up form asking name and age, display a welcome message only if age is valid.

Tip: Combine input.value, event listeners, and validation for full interactivity.

Check: Browser prevents invalid input and displays correct messages. This completes Week 13.
