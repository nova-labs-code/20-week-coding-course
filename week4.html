Week 4: CSS Layouts

Welcome to Week 4! This week we focus on **layout techniques** to control how elements appear on your page. You will learn **block vs inline, display property, Flexbox basics, alignment, positioning, and spacing**. Lessons follow “Learn → See → Do → Check” and now include more examples, exercises, and tips for each day.

Day 1: Block vs Inline Elements

Learn: HTML elements have default display types. Block elements like <div>, <p> occupy the full width and start on a new line. Inline elements like <span> or <a> occupy only the width of their content and do not force line breaks.

See:

<div>I am a block</div>
<span>I am inline</span><span>Me too</span>

Do:

  • Create a block element (div) and several inline elements (span, a) in your page.
  • Experiment with nesting inline elements inside block elements.
  • Try adding background colors or borders to see how each element behaves.

Check: Open in a browser and notice how block elements stack vertically while inline elements sit next to each other.

Tip: Understanding block vs inline is the foundation for more advanced layout techniques like Flexbox and Grid.
Day 2: Display Property

Learn: The CSS display property lets you override default element behavior. Key values include block, inline, inline-block, flex, grid, and none.

See:

p { display: inline-block; background: #ddd; padding:5px; }
div { display: none; }

Do:

  • Set a paragraph to inline-block and notice that it behaves like a block element but remains inline.
  • Hide a div using display:none and then revert it to block.
  • Try different display values like flex or grid on a container to see their effects.

Check: Observe in the browser how elements appear, disappear, and align differently based on display settings.

Tip: Use display:none carefully; the element is completely removed from layout and not just hidden.
Day 3: Introduction to Flexbox

Learn: Flexbox is a layout model that distributes space along a container. It is perfect for one-dimensional layouts, horizontal or vertical, with precise alignment and spacing control.

See:

<div class="flex-container">
  <div>Box 1</div>
  <div>Box 2</div>
  <div>Box 3</div>
</div>

<style>
.flex-container {
  display: flex;
  gap: 10px;
  background: #f0f0f0;
  padding: 10px;
}
.flex-container div {
  background: #ddd;
  padding: 20px;
}
</style>

Do: Create a flex container with 3-4 boxes. Adjust the gap between them and experiment with box widths.

Check: Open in a browser. Boxes should line up horizontally with spacing controlled by gap.

Tip: Flexbox simplifies horizontal and vertical alignment without floats or complicated margins.
Day 4: Flex Direction & Justify Content

Learn: Flexbox allows controlling direction with flex-direction (row | column) and horizontal alignment with justify-content (flex-start, center, space-between, space-around).

See:

.flex-container {
  display: flex;
  flex-direction: column;
  justify-content: center;
  height: 200px;
}

Do:

  • Change flex-direction between row and column.
  • Experiment with justify-content: center, flex-start, space-between, and space-around.
  • Add background color to container to visualize alignment.

Check: Open in a browser. Boxes should stack vertically or horizontally and align according to your settings.

Tip: Use container height to clearly see vertical alignment effects.
Day 5: Align Items & Align Self

Learn: Flexbox aligns items vertically using align-items for all children and align-self for individual elements.

See:

.flex-container { align-items: center; height: 200px; }
.flex-container div:nth-child(2) { align-self: flex-end; }

Do:

  • Center all boxes vertically with align-items.
  • Change one box’s alignment using align-self.
  • Combine with justify-content to see full layout control.

Check: Observe vertical positioning in the browser.

Tip: Align-self overrides align-items for individual boxes.
Day 6: Position Property

Learn: CSS position lets you place elements precisely: static (default), relative, absolute, fixed, sticky.

See:

div.relative { position: relative; top: 10px; left: 20px; }
div.absolute { position: absolute; top: 50px; left: 50px; }
div.fixed { position: fixed; bottom: 0; right: 0; }

Do:

  • Create one relative, one absolute, and one fixed div.
  • Move them around using top, left, right, bottom.
  • Observe how absolute positioning depends on nearest positioned ancestor.

Check: Open in a browser and experiment with combinations.

Tip: Use fixed position for headers, footers, or sticky buttons that remain visible on scroll.
Day 7: Review & Mini Layout Page

Learn: Combine all layout skills learned this week to structure a professional profile page. Apply block/inline understanding, display, Flexbox, alignment, and positioning. Do:

  • Create a main container for your profile sections.
  • Use Flexbox to layout your profile picture, bio, and hobbies horizontally.
  • Align items vertically and horizontally for visual balance.
  • Add positioning for a fixed footer or floating button.
  • Experiment with spacing, padding, and margins to avoid overlapping.

Check: Open the page in a browser and ensure layout is clean, sections are aligned, and the page looks organized. Test responsiveness by resizing the window.

Tip: Combining display, Flexbox, and positioning gives powerful control over your page layout without using floats.
