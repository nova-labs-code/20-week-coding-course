Week 9: JavaScript Basics – Variables, Data & Console

This week introduces **JavaScript fundamentals** to make your pages interactive. Each day has detailed explanations, code examples, exercises, and tips to practice using the browser console or <script> tags in your HTML.

Day 1: Introduction to JavaScript & Console

Learn: JavaScript is a programming language for dynamic web behavior. You can test your code instantly using console.log(), which prints messages in the console for debugging and checking values.

See:

console.log("Hello, world!");

Do: Open your browser console (F12 or right-click → Inspect → Console) and type: console.log("My name is [Your Name]"). Press Enter.

Check: The console should display your message.

Tip: Use console.log frequently while coding to check variables and outputs.
Day 2: Variables (let, const, var)

Learn: Variables store values. let is for changeable variables, const for constants, and var is older but still works. JavaScript is case-sensitive and variable names cannot start with numbers.

See:

let age = 15;
const name = "Elisha";
var hobby = "coding";
console.log(name, age, hobby);

Do: Create 3 variables for your favorite color, food, and game, then log them.

Check: Browser console shows your variable values correctly.

Tip: Use const whenever you don’t plan to change a value. It prevents accidental reassignments.
Day 3: Data Types & typeof

Learn: JavaScript has different data types: string, number, boolean, object, undefined, and null. Use typeof to check the type of a variable.

See:

let score = 100;
let isOnline = true;
console.log(typeof score);  // number
console.log(typeof isOnline); // boolean

Do: Create 2 variables of different types, then log their types with typeof.

Check: Console shows the correct data types.

Tip: Always check types before performing operations to avoid errors.
Day 4: Basic Arithmetic

Learn: JavaScript supports arithmetic operations: addition (+), subtraction (-), multiplication (*), division (/), modulus (%). You can perform calculations directly or store results in variables.

See:

let x = 10;
let y = 3;
console.log(x + y); // 13
console.log(x % y); // 1

Do: Calculate your age next year or the remainder when dividing two numbers.

Check: Verify results in the console.

Tip: The modulus (%) operator is useful for checking even/odd numbers or cycles.
Day 5: Strings & Concatenation

Learn: Strings are text values. Combine them using the + operator or template literals (backticks ``) for easier formatting.

See:

let firstName = "Elisha";
let lastName = "Samson";
console.log("Hello " + firstName + " " + lastName);
console.log(`Hello ${firstName} ${lastName}`);

Do: Create a greeting message using both concatenation and template literals.

Check: Console shows correct greetings with your name.

Tip: Template literals also support multi-line strings for easier formatting.
Day 6: User Interaction with prompt

Learn: Use prompt() to ask the user for input. Returned value is a string, so convert with Number() if needed.

See:

let age = prompt("Enter your age:");
console.log("You are " + age + " years old.");

Do: Prompt for your favorite hobby and log it to the console.

Check: Enter a value and verify the console displays it.

Tip: Use alert() or console.log() to show outputs to users during testing.
Day 7: Review & Mini JS Console Exercise

Learn: Combine variables, types, arithmetic, strings, and prompt to create small interactive console programs.

Do: Write a program that:

  1. Asks for the user's name and age using prompt().
  2. Calculates next year’s age.
  3. Logs a greeting message like: "Hello [Name], next year you will be [age+1]."
  4. Try adding your favorite hobby to the message for extra practice.

Check: Open console to ensure your program works correctly. Test with different inputs to see dynamic results.

Tip: Practice small scripts daily. The console is your playground for testing JavaScript logic safely.
