Week 3: CSS Basics

Welcome to Week 3! This week focuses on **CSS styling** to make your HTML pages visually appealing and organized. You will learn **inline styles, internal CSS, classes & IDs, colors, fonts, spacing, borders, and more**. Each day follows the “Learn → See → Do → Check” method, and lessons are longer with extra examples and exercises for better understanding.

Day 1: Introduction to CSS & Inline Styles

Learn: CSS (Cascading Style Sheets) adds visual style to your HTML elements. Inline styles are applied directly to elements using the style attribute. Inline styles override default styles but are not recommended for large projects.

See:

<p style="color:blue; font-size:18px; font-weight:bold;">Blue bold text</p>
<h1 style="color:red; font-family: 'Courier New', monospace;">Hello World</h1>

Do: Add inline styles to headings and paragraphs on your profile page. Change text color, font size, and font family for each element. Try bold or italic styles for emphasis.

Check: Open in a browser and confirm that styles appear correctly. Experiment with different font families, colors, and sizes to see the effect.

Tip: Inline styles are quick for testing, but for consistency across multiple pages, use internal or external CSS.
Day 2: Internal CSS & Selectors

Learn: Internal CSS is written inside a <style> tag in the HTML <head>. Selectors let you target specific elements by tag, class, or ID.

See:

<style>
  h1 { color: darkblue; font-size: 28px; }
  p { font-family: Arial; font-size:16px; line-height:1.8; }
  ul li { margin-bottom:5px; }
</style>

Do:

  • Add an internal <style> block in your page.
  • Style all headings, paragraphs, and lists using tag selectors.
  • Experiment with line-height, font sizes, and colors to make content readable.

Check: Verify in the browser that styles are applied. Try changing selector properties and see how the page updates.

Tip: Internal CSS keeps styling in one place and is easier to manage than inline styles.
Day 3: Classes & IDs

Learn: Classes (class) and IDs (id) allow more precise styling. IDs are unique and used once per page, while classes can be reused multiple times.

See:

<h2 class="section-title">About Me</h2>
<p id="bio">Hello!</p;

<style>
  .section-title { color: green; font-size:20px; }
  #bio { font-style: italic; color: #555; }
</style>

Do:

  • Assign classes to section headings like “Hobbies” or “Skills”.
  • Assign an ID to your main paragraph and style it differently.
  • Try using multiple classes on one element for combined effects.

Check: Open in a browser to see if each element has the correct style. Try changing colors and fonts to compare results.

Tip: Use descriptive class and ID names for clarity, e.g., class="profile-heading" or id="main-bio".
Day 4: Colors & Backgrounds

Learn: CSS lets you style text, backgrounds, and even borders. You can use color names, HEX codes, RGB, or HSL. Background colors improve readability and highlight sections.

See:

p { color: #333333; background-color: #f0f0f0; padding:10px; }
h1 { color: darkred; }
div.highlight { background-color: yellow; }

Do:

  • Set background colors for different sections of your profile page.
  • Change paragraph text color to complement the background.
  • Highlight important information with a colored div.

Check: Open in a browser to ensure text is readable against backgrounds. Adjust colors if needed for clarity and contrast.

Tip: Use contrasting colors for text and background to make content easy to read.
Day 5: Fonts & Text Alignment

Learn: CSS controls font type, size, weight, and alignment. You can use multiple fonts as fallback, adjust alignment with text-align, and emphasize with font-weight.

See:

p { font-family: 'Courier New', monospace; font-size:16px; text-align:center; }
h2 { font-weight:bold; text-align:left; font-family: 'Georgia', serif; }

Do:

  • Apply different fonts to headings and paragraphs.
  • Experiment with center, left, and right alignment for different sections.
  • Try bold, italic, or lighter font weights.

Check: Open in a browser to see how font choices and alignment affect readability and page layout.

Tip: Always provide fallback fonts for better compatibility across browsers.
Day 6: Borders, Padding & Margins

Learn: Borders define element edges, padding adds space inside, and margin adds space outside. This improves spacing, readability, and structure.

See:

p { border: 1px solid #ccc; padding: 15px; margin: 20px 0; border-radius:5px; }
div.section { border:2px dashed #1e90ff; padding:10px; margin-bottom:15px; }

Do:

  • Add borders around paragraphs and sections.
  • Experiment with different border styles: solid, dashed, dotted, double.
  • Adjust padding inside elements and margin outside elements.

Check: Open in a browser to observe how spacing and borders affect layout. Ensure elements do not overlap and text remains readable.

Tip: Use border-radius to make rounded corners for a modern look.
Day 7: Review & Mini Styled Page

Learn: Combine everything from Week 3 to style your Week 1/2 profile page. Use internal CSS, classes, IDs, colors, fonts, spacing, and borders to create a visually organized page.

Do:

  • Style all headings and paragraphs consistently.
  • Add colored backgrounds to sections.
  • Use classes and IDs to apply unique styles.
  • Include borders, padding, and margins for spacing.
  • Check readability on different screen sizes.

Check: Open the page in a browser. Confirm that all styles are applied correctly, text is readable, sections are visually distinct, and overall layout looks professional.

Tip: Review your CSS for repeated code. Consider using classes to reduce redundancy and keep your code clean.
